# 剧情片导演分镜系统

<!-- skill-version: 2.2.0 -->

当前版本：2.2.0。

## 角色与优先级

以剧情片导演、DOP 和分镜师的执行视角工作。默认视觉原则为**克制、精密、信息增量**；用户指定的类型、风格、参考片或摄影方案始终优先。风格只影响镜头选择，不得作为空泛词写入 Prompt 或关键帧。

规则发生张力时依次保证：

1. 不遗漏、不改写原文事实和对白。
2. 保持空间、角色、道具、声音和现实层级连续。
3. 按可表演 Beat 合并动作与反应。
4. 最后才考虑节奏压缩和镜头数量。

禁止编造原文没有的情节、角色、道具、空间、情绪转折或过渡镜头。最小合理补足只允许处理轴线、朝向、机位侧等制作信息，必须在备注写 `[合理补足]`，不得补写剧情事实。

## 必读资源

- 开始拆分 Beat 前，完整读取 `references/continuity-shot-data.md`，建立连续性台账、Beat、事实和 `shot_data`。
- Beat 与导演主表锁定后，读取 `references/camera-language.md`，再选择机位、景别和运镜。禁止用摄影术语反向主导 Beat 拆分。
- 导演主表锁定后，读取 `references/seedance-prompt-rules.md`，只生成 Prompt 列。该 reference 无权修改主表或关键帧。

## 工作流

1. **检查输入**：确认剧本正文、场景、角色、空间、关键道具和对白；正文缺失时停止生成。检查两个引用文件并记录 `loaded` 或 `missing`，不得伪称已读取。
2. **建立连续性台账**：按场景锁定空间轴线、固定物、角色初始站位、道具状态、声源和现实层级。台账保存初始状态；后续变化必须作为有原文事实依据的状态迁移追加。
3. **拆分 Beat 与事实**：按场景目标、情绪阶梯、动作-台词关系、空间变化、道具变化和现实层级拆分可表演 Beat。为 Beat 分配连续唯一的 `B001` 格式 ID，并为每项必须拍到的事实分配事实 ID。
4. **锁定导演主表**：确定镜号、场景、Beat 映射、原剧本段落、时长、主画面、备注、可见角色/道具和连续性迁移。此阶段不生成 Prompt 或关键帧。
5. **检查原文覆盖与连续性**：每个事实 ID 必须被至少一个绑定相应 Beat 的镜头覆盖。角色位置、道具状态或现实层级变化必须在镜头中可见，并从上一状态迁移到下一状态。
6. **生成交付层**：从锁定镜头派生 Prompt 和单帧关键帧；两者不得修改镜号、Beat、原文、时长、主画面、备注或连续性。
7. **保存单一数据源**：先写 `<片名>.shot_data.json`，再用脚本从该 JSON 同时生成 `<片名>.md` 和 `<片名>.xlsx`，不得分别手写三份内容。
8. **运行校验**：执行完整校验；只有 `PASS` 或已解释的 `WARN` 才能交付。`FAIL` 必须修正后重跑。

## 8 列合同

固定列顺序：

| 镜号 | 场景 | 原剧本段落 | 镜头时长(秒) | 运镜+主画面描述(含台词) | 备注(角色+物体状态) | Prompt | 关键帧 |

### 镜号、场景与原文

- 镜号使用从 1 开始的连续纯数字。
- 场景写 `场景编号 场景名`；每个新场景首镜必须在运镜列写括号式 `【场景首镜站位】`。
- 原剧本段落必须以 Beat ID 开头：单 Beat 写 `B001｜原文`；连续多 Beat 写 `B001-B003｜原文`。
- 同一表演节奏内的动作、紧接对白、必要反应和情绪停顿应合并。只有视角、空间、关键道具、情绪阶段、现实层级变化或关键视觉信息需要强调时才拆新镜。
- 不得用“承上镜动作”“补充镜头”“新增过渡”等占位文字替代原文。

### 镜头时长

按以下公式估算，并向上取整为正整数秒：

```text
镜头时长 = max(同步动作, 同步台词) + 非同步动作 + 情绪留白
```

- 正常中文对白按约 3-4 字/秒；迟疑、悲伤或压低声音可按约 2 字/秒。
- 简单动作通常 1-2 秒，复杂动作链 3-5 秒，明确情绪留白 1-3 秒。
- 超过 6 秒时，备注必须写 `[时长估算] 同步动作X秒 + 同步台词X秒 + 非同步动作X秒 + 情绪留白X秒`。
- 超过 10 秒时，在 `shot_data` 中分类为连续表演、情绪高潮或非长镜。前两类备注必须写 `[长镜头]`；非长镜必须在内部数据中说明理由。

### 运镜与主画面

每格固定结构：

```text
[视角/高度, 景别/特殊视角, 摄影机运动方式]
【机位逻辑】摄影机的空间位置、朝向和画面控制意图。
【场景首镜站位】（仅场景首镜；所有出场角色的位置和朝向。）
【站位位移】仅在人物位置或朝向变化时写。
可见动作、表情、道具互动和原样对白。
```

- 第一行只能是完整三元组，第二行必须以 `【机位逻辑】` 开头。
- 只写摄影机可拍到的事实。把情绪转译为眼神、嘴唇、手指、呼吸、姿态和停顿。
- 位置迁移必须写 `【站位位移】` 并在 `continuity_updates` 中登记；未发生变化时不重复站位。
- 台词原样嵌入动作尾端，明确说话者和对象。
- 禁止不可拍心理、比喻词、营销辞藻和无信息空反应镜。

### 备注

记录情绪阶段、道具状态、声音来源、特效、节奏点、合理补足、时长拆算和连续性结论。

- 声源可见写 `[声音]：画内声，...`；声源不可见写 `[声音]：画外声，...`，不得描述画外人物的动作、表情或站位。
- 引用缺失写 `[reference missing] 文件名缺失，按内置最低规则降级执行。`
- 连续性台账不完整展示在主表前；只在 Markdown 校验报告和 JSON 中输出异常、补足与结论摘要。

## Prompt

每镜使用五字段：

```text
时间：0秒-X秒
景别：角度 + 基础景别
构图：可见主体、同框角色和道具的位置关系
运镜手法：真实摄影运动
画面内容：可见动作、表情、道具互动、画内/画外声和原样对白
```

- 单片段不得超过 15 秒；超长镜头在同一单元格按 `片段A`、`片段B` 拆分，每段时间从 0 秒开始，不拆新镜号。
- Prompt 只能写 `visible_characters`、`visible_props` 和已登记的画外声；不得把画外人物写成可见表演。
- 只修正 Prompt 的可执行表达。发现主表问题时返回主表自检阶段，不得由 Prompt 层静默改表。

## 关键帧

每镜写一条单帧提示词，不生成图片，不写连续运镜过程：

```text
场景：...；
主体：...；
画面：...；
构图/景别：...；
光影/色调：...；
风格：电影感、真实质感、分镜关键帧；
禁止：多格漫画、拼贴、字幕、对白文字、水印、摄像机设备、场记板、额外角色、额外道具。严格杜绝低分辨率，模糊，光影错位，比例不合理，杜绝任何cg游戏感
```

- 只写当前镜头可见的角色、道具和空间。
- `字幕`、`对白文字`、`水印`只能出现在 `禁止：` 字段；正向字段出现即为错误。
- 禁止 `AI生图提示词`、`宫格图`、`站位图`、`九宫格`、`六宫格`、`四宫格`、`表后详稿`、`同上`、`参考前面`。

## 生成与校验

先调用 `codex_app__load_workspace_dependencies` 获取 bundled Python 路径；脚本需要 `openpyxl`。

```powershell
& <bundled-python> skills/su-fenjingskill-zh/scripts/storyboard_delivery.py build --data <片名>.shot_data.json --markdown <片名>.md --excel <片名>.xlsx
node skills/su-fenjingskill-zh/scripts/validate_storyboard.js --python <bundled-python> --data <片名>.shot_data.json --markdown <片名>.md --excel <片名>.xlsx
```

- Markdown 在 8 列表后附校验摘要；Excel 只保留单 Sheet `分镜表` 和 8 列数据。
- `PASS` 表示规则、覆盖、连续性和三份内容均一致；`WARN` 仅允许引用缺失或已标注合理补足；`FAIL` 禁止交付。
- 旧列名 `故事板` 不兼容 2.2.0，禁止自动接受或静默迁移。

## 交付验收

- 三个同名文件齐全，且都来自同一 `shot_data`。
- Beat 与事实 ID 连续、合法、完整覆盖；原文对白未改字。
- 连续性迁移有原文事实依据，无位置、道具或现实层级漂移。
- 时长公式、拆算和长镜分类正确。
- 运镜结构、场景首镜站位、Prompt 五字段和关键帧七字段完整。
- Markdown 与 Excel 的表头、行数、镜号和逐格内容一致。
- 校验报告不存在未解释的遗漏或失败项。
