# Image2 九宫格分镜提示词

## 版本

<!-- skill-version: 1.11.0 -->

当前版本：1.11.0。后续每次优化前，先保留当前版本；优化后递增版本号，并使用 Git 标签 `su-image2-storyboard-grid-zh-v{版本号}` 保存历史版本。

把 `su-fenjingskill-zh` 的分镜主表、局部镜号、某场戏，或用户提供的纯剧情文本，整理成适用于 Image2 的固定 3x3 九宫格黑白分镜图提示词。

本 skill 是独立下游交付工具。它只读取主表或剧情，不回写、不修订、不替换 `su-fenjingskill-zh` 的任何规则。

## 核心边界

- 不修改 `su-fenjingskill-zh`。
- 不回写主表。
- 不改变镜号、场景、原剧本段落、镜头时长、运镜主画面、备注、Prompt 列或故事板列。
- 不替换 `su-fenjingskill-zh` 现有“每镜单帧故事板提示词”规则。
- 不把九宫格、多宫格、站位图或 ZIP 交付规则写入主表。
- 如果需要和主表关联，只在本 skill 的 Markdown 中写“来源镜号/镜号范围”。

## 默认交付

默认生成一个 Markdown 提示词文件，而不是只在聊天中输出长文本。

文件名建议：

```text
image2_storyboard_grid_prompts_[项目名或场次名].md
```

如果用户没有指定保存位置，保存到当前任务的用户可见输出目录；如果环境没有输出目录，则保存在当前工作目录下的 `outputs/` 或用户指定目录。

Markdown 文件按镜号、镜号范围或场景组织，每段都是一条可复制到 Image2 的九宫格提示词。

## 输入优先级

按以下优先级处理输入：

1. 用户本轮明确要求。
2. 用户提供的美术资产或参考图。
3. `su-fenjingskill-zh` 主表中的镜号、场景、原剧本段落、运镜主画面、备注、Prompt、故事板列。
4. 纯剧情文本。

用户提供主表时，主表是剧情事实来源。用户只提供剧情时，允许从剧情推演九个画面节点，但不得新增剧情事实。

## 完整阅读与空间轴线规则

规划任何九宫格之前，必须先完整阅读用户提供的全部剧情、分镜表、截图文字或可见表格内容，不得只截取局部句子直接规划宫格。

有分镜时：

1. 先完整阅读剧情和分镜内所有可见镜头。
2. 根据场景建立空间概念，明确主要场景边界、入口出口、远近层次、角色初始位置、车辆/动物/道具初始位置。
3. 锚定方向和摄影机轴线，写清画面左/右、前/后、近/远、主轴方向、主要反打方向。
4. 最后完全按照分镜内的规划来切分九宫格；不得重新发明镜头顺序，不得把分镜没有安排的动作塞进九格。

没有分镜、只有剧情文本时：

1. 先完整阅读剧情全文，确认事件起点、动作推进、人物关系和结尾状态。
2. 根据场景建立空间概念，明确入口出口、主通道、角色/动物/车辆/道具初始位置和可见边界。
3. 锚定方向和摄影机轴线，先确定视觉连续规则，再规划九宫格。
4. 九宫格只能从剧情文本中推导可见节点；不得为了画面丰富新增剧情事实。

空间锚定后，场景、人物、动物、车辆、道具等都必须以初始坐标为锚点，禁止漂移。

- 只有剧情或分镜明确交代位移时，才允许位置变化。
- 允许的位移必须写清起点、路径、终点，例如“A 从 B 的左侧走到 B 的右侧”“车辆从画面左边驶入画面中心，停下，然后重新启动从画面右边驶出”。
- 后续宫格内没有被剧情、分镜或镜头规划特别提及的人物、动物、车辆、道具，严禁乱入。
- 不允许同一车辆、同一道具或同一人物在相邻格中无理由换边、消失、出现、改变归属或改变运动方向。
- 整个九宫格必须形成完整、流畅、可追踪的视觉顺序，观众应该能看懂人物、车辆和道具从哪里来、到哪里去、何时停留、何时离开。

## 室内平面布局锁定规则

室内场景进入房间、病房、书房、卧室、客厅、教室、办公室、店铺、车厢等封闭或半封闭空间后，必须先建立一张房间平面布局。家具和固定陈设不能只写成“背景可见”，必须绑定到门、窗、墙、床、桌、沙发、柜体、走廊入口等固定参照物。

必须先定义：

- `房间边界`：门所在墙、窗所在墙、床/桌/沙发所在墙、主要通道和可行走区域。
- `主家具坐标`：床、沙发、桌子、梳妆台、书柜、柜台、讲台、病床、车座等相对门/窗/墙的位置。
- `固定陈设坐标`：照片、镜子、台灯、绿植、相册、梳子、杯子、监护仪等与主家具或墙面的从属关系。
- `机位转换`：从门口看、从房内反拍门口、从床侧看、从窗边看、从高位俯拍时，家具在画面中的投影如何变化。

规则：

- 角色进入房间后的第一个室内全景或主观全景必须承担“平面图建立”功能，清楚交代门、窗、床/桌/沙发、关键道具之间的相对位置。
- 后续近景、特写、反打、俯拍、主观视角都必须从这张平面图推导，不能每格重新布置家具。
- 家具可以因机位变化出现在画面左/右/前/后，但其世界坐标、所靠墙面、与门窗的距离关系不能改变。
- 固定墙面陈设只能留在同一面墙；照片、镜子、窗、门、柜子不得在不同格中换墙、换边、翻转或漂移。
- 如果某格为特写，只能裁切空间，不能通过特写暗示家具已经换位置；需要在提示词里写清“同一家具局部”“同一台面局部”。
- 如果剧情没有明确搬动、拿起、放回、关门、开窗、坐下、起身等动作，家具和固定陈设不得发生位置或状态变化。

## 俯视平面图锚定规则

当用户提供室内俯视图、平面图、站位图、简笔画平面图、手绘草图、鸟瞰图，或文字说明中出现“门、窗、床、桌、沙发、梳妆台、人物站位、机位箭头”等平面布局信息时，必须把它作为空间锚定的最高优先级来源。

俯视图可以非常粗糙，允许是简笔画、线框、方块、箭头或手写标注；不要求美术精细。它只负责锁定“对象在哪里、朝向哪里、机位在哪里”，不负责成片风格。

优先级固定为：

1. 俯视平面图、简笔画平面图、站位图。
2. 用户给出的机位图、机位箭头、镜头方向说明。
3. 场景透视参考图、概念图、剧照式场景图。
4. 角色、道具、美术风格参考图。
5. 纯文字补充说明。

如果透视参考图与俯视图冲突，以俯视图为准；如果多张透视参考图之间冲突但没有俯视图，只输出资产映射确认，不生成正式提示词。

必须遵守：

- 不把俯视图画进九宫格成片，除非用户明确要求输出平面图页面。
- 将俯视图转写成具体世界坐标：哪面墙、哪个角落、距离哪个门/窗/家具、朝向哪里。
- 俯视图中的机位箭头必须转写成机位编号，例如 `机位 A`、`机位 B`、`机位 C`。
- 每个九宫格画面必须绑定一个机位编号；不能让模型在没有机位依据的情况下自由新建角度。
- 如果某格没有明确机位，优先复用最近的已建立机位或从剧情动作推导最近合理机位，不能新增会破坏平面布局的新机位。

## 俯视平面图锚定表

生成 Markdown 时，只要用户提供了俯视图、平面图、站位图或简笔画平面图，必须在 `资产控制` 后加入 `俯视平面图锚定表`。这张表只写在 Markdown 里，不能要求生成到图片中。

俯视平面图锚定表固定列：

```markdown
| 对象/区域 | 平面图标记或识别点 | 世界坐标/所靠墙面 | 朝向/开口方向 | 与其他对象关系 | 约束 |
|---|---|---|---|---|---|
| ... | ... | ... | ... | ... | ... |
```

写法要求：

- `平面图标记或识别点` 必须写用户图里的可辨识标记，例如“门框线”“床矩形”“A 箭头”“窗线”“梳妆台小矩形”。
- `世界坐标/所靠墙面` 必须从俯视图推导，不能只写“画面左/右”。
- `朝向/开口方向` 必须写门开向、床头方向、桌面朝向、人物面向、车辆车头方向或机位箭头方向。
- `与其他对象关系` 必须写清相邻、相对、遮挡、从属和可行走通道。
- `约束` 必须写明不得漂移、换墙、镜像、复制、调头或越过边界。

## 机位编号绑定规则

同一场景九宫格必须先建立可复用机位，再把每格绑定到机位编号。机位是“摄影机在俯视平面图或室内平面图中的位置 + 镜头朝向”，不是泛泛的景别。

规则：

- 机位编号用 `A/B/C/D...`，每个编号只能对应一个固定摄影机位置和镜头朝向。
- 一个机位可以服务多个格子；同机位下家具投影应保持一致，只允许景别缩放或局部裁切。
- 新增机位必须说明它在平面图中的位置和朝向；不得为了某一格好看临时新增违背布局的机位。
- 特写格也要绑定机位，例如“机位 B 的照片局部特写”“机位 C 的同一梳妆台台面局部”。
- 如果反打导致屏幕左/右变化，必须在机位表中说明这是同一世界坐标的投影变化。

## 机位编号表

生成 Markdown 时，来自同一场景连续九宫格且需要稳定空间时，必须在 `摄影机朝向表` 前加入 `机位编号表`。

机位编号表固定列：

```markdown
| 机位 | 平面图位置 | 镜头朝向 | 服务格 | 应见背景 | 禁止背景 | 继承说明 |
|---|---|---|---|---|---|---|
| A | ... | ... | ... | ... | ... | ... |
```

写法要求：

- `平面图位置` 必须绑定俯视图或室内平面布局中的固定参照物。
- `镜头朝向` 必须写清朝哪面墙、哪个门窗、哪件主家具或哪个动作中心。
- `服务格` 必须列出格号；每格只能绑定一个主机位。
- `应见背景` 和 `禁止背景` 必须和该机位方向一致，不能互相矛盾。
- `继承说明` 必须写清同一机位下哪些家具和道具投影必须保持一致。

## 室内平面布局表

生成 Markdown 时，室内场景必须在 `主体建筑朝向表` 后、`门槛边界表` 前加入 `室内平面布局表`。这张表只写在 Markdown 里，不能要求生成到图片中。

室内平面布局表固定列：

```markdown
| 空间/家具/陈设 | 固定参照物 | 世界坐标/所靠墙面 | 与其他对象关系 | 可见格投影 | 禁止漂移 |
|---|---|---|---|---|---|
| ... | ... | ... | ... | ... | ... |
```

写法要求：

- `固定参照物` 必须绑定到门、窗、墙、床、桌、沙发、柜体、楼梯口等稳定对象，不能只写“画面左侧/右侧/远处”。
- `世界坐标/所靠墙面` 必须说明该家具或陈设位于哪一面墙、哪个角落、哪件家具上，或与门窗之间的前后左右关系。
- `与其他对象关系` 必须写清床与梳妆台、门与窗、照片与床头、相册与沙发、台灯与桌面等从属和相邻关系。
- `可见格投影` 必须说明同一家具/陈设在不同机位下是全貌、局部、反打背景、俯拍台面还是画外。
- `禁止漂移` 必须写清不得换墙、换边、换到另一件家具上、镜像、复制成多个、在特写中变成另一套家具。

## 前序宫格布局继承规则

同一场景的后序宫格必须继承前序宫格已经建立的空间布局。`空间概念` 和 `室内平面布局表` 只定义世界坐标；真正生成每一格时，必须按前一格或最近一个全景格的可见布局继续推导，不能把每一格当成重新建立空间的独立画面。

必须区分：

- `世界坐标`：门、窗、床、桌、车、人物、道具在场景中的真实位置。
- `屏幕位置`：某个机位下，世界坐标投影到画面后的左/右/上/下位置。
- `继承锚点`：上一格已经清楚出现并且下一格仍在同一空间内必须沿用的门、窗、墙、家具、道具、人物站位。

规则：

- 后序格必须明确“继承自第几格”或“继承自最近的空间建立格”，并写清继承哪些锚点。
- 反拍、侧拍、俯拍会改变屏幕左/右，不能要求所有机位都保持同一屏幕左右；必须写成“同一世界坐标的对应投影”，不能把“从门口看画面左”直接套到反拍画面里。
- 如果机位变化会让某些家具不该可见，必须裁掉或写画外，不要为了证明布局而强行把所有家具塞进画面。
- 近景和特写只能继承局部锚点，例如“同一北墙结婚照局部”“同一东墙梳妆台台面局部”；不得在特写中重新生成一套房间背景。
- 后序格除非剧情明确交代，不得新增前序格没有建立过的同类固定结构，例如第二张床、第二个窗、第二套梳妆台、另一面照片墙。
- 如果一个后序格需要同时看到多个相距很远的锚点，必须检查该机位是否合理；不合理时优先保留主动作和最近锚点，其他锚点画外。

## 前序宫格布局继承表

生成 Markdown 时，同一场景连续九宫格必须在 `室内平面布局表` 后、`门槛边界表` 前加入 `前序宫格布局继承表`。这张表只写在 Markdown 里，不能要求生成到图片中。

前序宫格布局继承表固定列：

```markdown
| 格 | 继承自 | 必须继承的空间锚点 | 本格新增/变化 | 本格可裁掉 | 禁止重建 |
|---|---|---|---|---|---|
| ... | ... | ... | ... | ... | ... |
```

写法要求：

- `继承自` 必须写明确格号；第一个空间建立格可写“建立格”，后序格必须写最近可继承的前序格。
- `必须继承的空间锚点` 必须是前序格已经建立的具体对象，不能写泛泛的“房间布局”。
- `本格新增/变化` 只能写剧情明确发生的动作、表情、视线、拿起/放回等变化。
- `本格可裁掉` 必须写清因为机位或景别变化可以不出现的锚点，避免模型为了展示全空间而重画布局。
- `禁止重建` 必须写清不得新增同类家具、不得换墙、不得镜像、不得把屏幕左右当成世界坐标。

## 对象足迹与朝向锁定规则

同一对象在多个格子中可见时，必须锁定它的世界坐标、占地足迹和朝向。`可见` 只表示该对象可以被拍到，不表示可以在每次出现时重新摆放。

必须锁定：

- `世界坐标`：对象相对固定参照物的位置，例如“院门外侧右方、右门柱外侧 1 个车身距离、车身不进入两门柱之间”。
- `占地足迹`：对象覆盖的空间范围，例如车辆四轮所在区域、人物站位、道具摆放点。
- `朝向`：车辆车头方向、人物身体朝向、动物头尾方向、道具正面朝向。
- `投影变化`：摄影机换机位后，对象在画面里的屏幕位置可以因透视变化，但必须由同一世界坐标和朝向投影出来，不能变成另一处停放或另一种朝向。

规则：

- 不允许用“画面右侧”“画面下方”“远处”单独定义对象位置；必须同时写世界坐标参照物。
- 同一车辆在多个可见格出现时，必须保持车头朝向、车尾方向、车身与道路/门槛的夹角一致。
- 高位远景、平视、车内窗外视角看到的同一车辆，必须是同一个停放点的不同投影，不得从门外右侧变成门内、从横停变成竖停、从车头朝主楼变成车头朝侧墙。
- 如果剧情没有明确“重新启动、调头、倒车、驶入、驶出、转弯、停下”，车辆不得改变世界坐标和朝向。
- 如果投影难以在生图中稳定，优先减少该对象在非必要格中的可见次数；不要让模型在多个大幅变化机位中反复重画同一对象。

## 对象足迹与朝向表

生成 Markdown 时，`门槛边界表` 后、`摄影机朝向表` 前必须加入 `对象足迹与朝向表`。这张表只写在 Markdown 里，不能要求生成到图片中。

对象足迹与朝向表固定列：

```markdown
| 对象 | 世界坐标锚点 | 占地足迹 | 固定朝向 | 可见格投影 | 禁止变化 |
|---|---|---|---|---|---|
| ... | ... | ... | ... | ... | ... |
```

写法要求：

- 同一对象只要在两个以上格子可见，就必须列入。
- `世界坐标锚点` 必须绑定固定参照物，不能只写屏幕位置。
- `固定朝向` 必须写清车头/身体/正面朝向。
- `可见格投影` 必须说明每个可见格里对象应由同一锚点如何被看见。
- `禁止变化` 必须写清不得位移、不得旋转、不得调头、不得换边、不得越过门槛。

## 门槛边界锁定规则

场景中只要存在门、院门、车门、房门、走廊入口、桥、台阶、洞口、病房门、街角、道路边线等“分隔内外/前后/上下/左右空间”的结构，必须先建立门槛边界。门槛边界是不可漂移的空间分界线，物体在哪一侧必须被锁定。

必须区分：

- `门外/外侧`：尚未进入门槛后的空间。
- `门内/内侧`：已经越过门槛进入内部通道、院内、房内或新空间。
- `门槛线上`：正处于通过边界的位置，必须由剧情或镜头明确说明。

规则：

- 车辆、人、动物、道具位于门外时，不能被画到门内，除非剧情明确交代“进入”“驶入”“穿过门”。
- “停在门口”“停在门边”“停在远处路口”不能自动等于进入门内；必须写清是在门外道路侧、门内通道侧，还是门槛线上。
- 高位远景和大远景也必须遵守门槛边界；不能因为俯视就把门外车辆画到门内道路上。
- 如果对象从门外进入门内，必须写清越过门槛的格号、运动路径和停留位置。
- 如果剧情没有明确对象穿越门槛，默认对象保持在初始侧，不得越界。

## 门槛边界表

生成 Markdown 时，`主体建筑朝向表` 后、`对象可见性表` 前必须加入 `门槛边界表`。这张表只写在 Markdown 里，不能要求生成到图片中。

门槛边界表固定列：

```markdown
| 边界 | 外侧/近侧 | 内侧/远侧 | 对象所在侧 | 允许跨越 | 禁止错乱 |
|---|---|---|---|---|---|
| ... | ... | ... | ... | ... | ... |
```

写法要求：

- 每个门、院门、房门、车门、洞口、走廊入口、桥、台阶、道路边线都要列入。
- `对象所在侧` 必须写清关键对象在哪一侧。
- `允许跨越` 只能写剧情明确发生的跨越。
- `禁止错乱` 必须写清不得把外侧对象画到内侧、不得把内侧对象画到外侧、不得无理由站在门槛线上。

## 摄影机朝向与背景锁定规则

机位不只决定景别，也决定画面背景。每个格子必须写清摄影机所在位置、镜头朝向、被摄主体朝向，以及画面背景属于角色前方还是后方。

基本规则：

- 摄影机在角色前方倒退拍角色正面时，镜头朝向角色来处；画面背景通常是角色身后/来路方向，不是角色前进目标方向。
- 摄影机在角色后方跟拍背影时，镜头朝向角色前进方向；画面背景通常是角色前方/目标方向。
- 主观视角必须以角色眼睛朝向为准；角色看向哪里，背景就应该是那个方向的空间。
- 高位远景必须写清是从起点回看、从终点回看，还是沿主轴俯视；不同高位方向会改变车、门、主楼在画面中的上下左右位置。
- 如果画面需要看到目标建筑在角色背后，必须说明摄影机位于角色目标侧并反拍，或说明角色回头；否则目标建筑不能无理由出现在角色背后。
- 同一格中，摄影机朝向和背景可见内容不能互相矛盾。

## 摄影机朝向表

生成 Markdown 时，`门槛边界表` 后、`对象可见性表` 前必须加入 `摄影机朝向表`。这张表只写在 Markdown 里，不能要求生成到图片中。

摄影机朝向表固定列：

```markdown
| 格 | 摄影机位置 | 镜头朝向 | 主体朝向 | 背景应见 | 背景禁见 |
|---|---|---|---|---|---|
| ... | ... | ... | ... | ... | ... |
```

写法要求：

- 每一格都必须列出。
- `背景应见` 必须由摄影机位置和镜头朝向推导，不能只凭剧情想象。
- `背景禁见` 必须明确容易乱入的对象、建筑或空间。
- 如果某格机位发生切换，必须说明切换后建筑、车辆、人物相对方向如何转换。

## 主体建筑朝向锁定规则

主体建筑、院门、走廊、楼梯、祭坛、洞口、主楼、店门等固定空间结构，必须在空间概念建立时锁定朝向。摄影机机位切换后，建筑可以呈现不同角度，但必须由初始锚定关系推导，不能让建筑本身换朝向、镜像、旋转或漂移。

必须先定义：

- 建筑正面朝向哪里。
- 入口、门、窗、柱、墙角、牌匾、台阶、屋顶轮廓等识别点分别在建筑的哪一侧。
- 摄影机位于建筑的正面、左前侧、右前侧、侧面、背面、高位或低位时，应该看到哪一面。
- 角色、车辆、道路、院门、主通道与建筑正面之间的方向关系。

机位切换规则：

- 摄影机从建筑正面切到左侧，画面应看到正面与左侧面；不得突然看到右侧面。
- 摄影机从建筑正面切到右侧，画面应看到正面与右侧面；不得突然看到左侧面。
- 摄影机退远或升高，只改变俯仰角和可见范围，不改变建筑朝向。
- 摄影机反打到背面时，必须说明这是“反向/背面机位”；建筑可见立面应从正面切换为背面或背侧关系，不能仍画成正面。
- 如果分镜没有明确反打或绕行，默认保持同一侧机位，不得无理由越轴到建筑另一侧。
- 主体建筑的入口、窗、柱、台阶、门洞等识别点在九格中必须保持归属和左右关系一致，不能在相邻格中无理由换边。

## 主体建筑朝向表

生成 Markdown 时，`空间概念` 后、`对象可见性表` 前必须加入 `主体建筑朝向表`。这张表只写在 Markdown 里，不能要求生成到图片中。

主体建筑朝向表固定列：

```markdown
| 建筑/空间结构 | 初始朝向锚点 | 识别点左右关系 | 机位切换对应可见面 | 禁止错乱 |
|---|---|---|---|---|
| ... | ... | ... | ... | ... |
```

写法要求：

- 每个主体建筑或固定空间结构都要列入，尤其是主楼、院门、店门、走廊、洞口、祭坛、病房门、楼梯等。
- `初始朝向锚点` 必须写清建筑正面朝向、入口朝向、主通道方向。
- `识别点左右关系` 必须写清门、窗、柱、墙、台阶、招牌、主楼轮廓等关键识别点在建筑左/右/前/后的位置。
- `机位切换对应可见面` 必须说明不同机位能看到正面、左侧面、右侧面、背面或高位平面关系。
- `禁止错乱` 必须写清不得镜像、不得左右互换、不得正反面混用、不得无理由越轴。

## 对象可见性锁定规则

空间锚定之后，还必须锁定关键对象在九格中的“可见/画外”状态。空间位置正确不等于可以出现在画面里；如果某格的镜头规划没有提到某个人、动物、车辆或道具，它默认不得进入该格画面。

必须区分：

- `可见`：该对象必须或允许出现在该格画面里。
- `画外`：该对象仍存在于空间中，但在该格镜头画面外，禁止画成远景、背景、小黑点、倒影或遮挡物。
- `不存在/未进入`：剧情中尚未出现或不属于该段，不得生成。

提示词中禁止使用会误导生成器的模糊说法：

- 不要写“在远后方”“仍在远处”“背景可见”“可隐约看到”，除非该对象在该格确实应该被看见。
- 如果对象在该格不该出现，必须写“完全画外、不可见、不要画出任何该对象或类似轮廓”。
- 对车辆尤其要写清：车是否可见、是否画外、是否禁止出现车身/车轮/车窗/车灯/车影/类似车辆形状。

对象可见性必须和 `空间概念`、`九格取舍表`、原始分镜事实三者一致：

- 如果 `空间概念` 或 `禁止漂移` 已经写明某对象在某格禁入，`对象可见性表` 不得把该对象列为可见。
- 如果原始分镜没有提到某对象在该镜头可见，默认该对象画外，除非该镜头的构图逻辑必须保留它作为空间锚点。
- 角色离开车辆、房间、门口、走廊入口等起点后，后续跟拍镜头默认进入角色的新空间；起点处的车辆、门口、房间内部默认画外，不能写成“远后方可见”。
- “车留在后方”“门口留在身后”“房间在背后”等描述只表示空间关系，不等于允许它们进入画面。若画面不需要看见，必须写“画外不可见”。
- 生成提示词前必须做一次内部冲突检查：同一对象同一格不能同时出现在 `可见格` 和 `画外格`，也不能一边写“禁入/排除”，一边在 Panel 描述里写“仍可见/在背景/在远处”。

## 对象可见性表

生成 Markdown 时，`空间概念` 后、`九格取舍表` 前必须加入 `对象可见性表`。这张表只写在 Markdown 里，不能要求生成到图片中。

对象可见性表固定列：

```markdown
| 对象 | 可见格 | 画外格 | 锁定说明 |
|---|---|---|---|
| ... | ... | ... | ... |
```

写法要求：

- 每个关键人物、动物、车辆、道具和场景锚点都要列入。
- `可见格` 必须明确写格号范围。
- `画外格` 必须明确写格号范围，并说明对象仍在空间中但镜头不拍到。
- 对容易乱入的对象，必须在 `锁定说明` 中写“禁止在画外格以远景、背景或类似轮廓出现”。

## 空间锚定表

生成 Markdown 时，`九格取舍表` 前必须加入 `空间锚定表`。这张表只写在 Markdown 里，不能要求生成到图片中。

空间锚定表固定格式：

```markdown
空间概念：
- 场景边界：...
- 方向锚定：...
- 摄影机轴线：...
- 初始坐标：...
- 允许位移：...
- 禁止漂移：...
```

写法要求：

- `方向锚定` 必须明确画面左/右、前/后、近/远与主要运动方向。
- `摄影机轴线` 必须说明主要视线轴或动作轴，后续格子不得无理由越轴。
- `初始坐标` 必须列出关键人物、动物、车辆、道具和建筑参照物的位置。
- `允许位移` 只能写剧情或分镜明确出现的移动。
- `禁止漂移` 必须明确哪些对象在未交代时不得乱入、换边、换位置或消失。

## 表格截图输入规则

当用户提供的是分镜表截图、Excel 截图或局部表格截图时，先把截图里可见信息人工转写成结构化来源，再生成提示词。不得只根据截图大意写一条泛化提示词。

必须先识别并保留：

- 场景号和场景名。
- 可见镜头数量和顺序。
- 每个镜头的主体、动作、景别、机位、运镜、站位、空间关系和关键物体。
- 人物在车内、车外、门口、道路、建筑之间的相对位置。
- 车辆、门、道路、建筑、手持物等连续道具的归属和状态。
- 截图中明确出现的对白或动作结果；不得把未出现的剧情补进九格。

如果截图模糊到无法判断镜头顺序、人物归属或关键空间关系，先向用户说明缺口并要求补图或原表，不生成正式提示词。

## 九格取舍表

生成 Markdown 时，`Image2 可复制提示词` 前必须加入一个简短的 `九格取舍表`。这张表只写在 Markdown 里，不能要求生成到图片中。

取舍表固定列：

```markdown
| 格 | 来源镜头 | 保留画面事实 | 取舍理由 |
|---|---|---|---|
| 1 | ... | ... | ... |
```

取舍规则：

- 来源镜头少于九个时，只能拆分原镜头里的可见阶段、不同景别、不同观察角度或关键反应来补足。
- 来源镜头多于九个时，保留最能表达空间关系、动作推进、情绪转折和结果状态的九个节点。
- 每一格都必须能追溯到一个或多个来源镜头。
- 不允许为了画面丰富新增原表没有的人、车、建筑、动作、道具或戏剧事件。
- 如果某格是由同一镜头拆分出来，取舍理由要说明它拆的是“建立空间”“动作开始”“反应特写”“结果状态”等可见阶段。

## 资产规则

用户提供美术资产或参考图时必须参考，并转成具体画面控制信息。

- 角色图控制：外貌、服装、体态、身份识别点。
- 俯视图/简笔平面图控制：对象世界坐标、所靠墙面、朝向、人物站位、移动路线、机位位置和镜头方向；优先级高于透视场景图。
- 场景透视图控制：空间结构、门窗、通道、家具、纵深、边界、环境基调；若与俯视图冲突，以俯视图为准。
- 道具图控制：形态、尺寸、材质、功能、被触碰或使用的方式。
- 站位图或尾帧控制：位置、朝向、距离、初始空间关系、上一段结束状态。
- 文字剧情控制：剧情事实、动作因果、对白原文、情绪变化、关键结果。

不要写“如图所示”“严格参考图片”“根据图片”等元话术。必须把图片信息转写成具体、可生成的画面描述。

当图片用途不清、编号无法对应、参考图之间冲突，或用户文字与站位图/尾帧/场景图冲突时，只输出资产映射确认并停止，不生成正式提示词。

确认前不生成 Markdown 提示词文件，不生成 PNG，不生成 ZIP。

- `室内平面布局表`：来自室内、房间、车厢、教室、书房、卧室、病房、客厅、办公室、店铺等场景时必须输出；必须明确门、窗、墙、主要家具、固定陈设和道具之间的平面关系、可见格投影和禁止漂移。

## 九格内容规则

## 角色出镜规则

九宫格的连续性目标是“同一身份不变”，不是“每格出现同一批角色”。

- 单人镜头：只写当前格实际出镜的角色；其他角色可通过视线方向、画外声或空间压力存在，但不要画进画面。
- 双人镜头：写清两人的朝向、距离、谁在前景/后景、谁正脸可见、谁侧脸或背影可见。
- 三人镜头：优先建立三人的空间关系；如果画面过载，保留核心两人，第三人可放在远处背景或下一格交代。
- 群像镜头：只重点跟踪 1 到 3 个核心角色，其余人写成背景人群或队列，不逐一刻画身份。
- 正反打或特写：不要为了“角色一致”把未出镜角色硬塞进画面。
- 不允许同一角色在同一格里被复制成两个人；角色离开画面后，除非剧情显示其回到画面，否则后续格不强行出现。
- 角色外貌、服装、体态和道具归属只在该角色出镜时保持一致。

## 画面风格

默认风格固定为：黑白铅笔素描 + 导演分镜草图 + 轻灰阴影。

生图提示词中必须主动压低成片质感，避免生成漂亮插画替代分镜：

- 使用“粗略导演分镜缩略图、黑白铅笔线稿、低精度面部、清楚站位、清楚空间关系、轻灰阴影”。

- 场景和站位优先级高于角色好看程度。
- 每格边框清楚，九格大小一致，不能生成单张连续长画卷。

- 不输出解释性总结到 Markdown 文件正文里。
- 不把内部推理节点、检查清单或修改说明写入可复制提示词。
- 不推荐其他模型或平台，除非用户明确询问。
- 不生成 JPG，除非用户明确要求。
- 不将 ZIP 作为默认提示词交付；ZIP 只用于用户追加生图后的图片交付。

## 输出前自检

在交付前静默检查：

1. 是否没有修改或要求修改 `su-fenjingskill-zh`。
2. 是否默认输出 Markdown 文件。
3. Markdown 是否按镜号、镜号范围、场景或剧情段组织。
4. 每段是否有一条完整可复制的 Image2 九宫格提示词。
5. 是否固定 3x3 九宫格，且整体图片必须是横向 16:9。
6. 是否每格也必须是横向 16:9。
7. `Image2 可复制提示词` 是否明确包含 `wide 16:9 canvas` 或 `wide horizontal 16:9 canvas`。
8. `Image2 可复制提示词` 是否明确包含 `rough black-and-white pencil storyboard sketch`。
9. 是否固定黑白铅笔素描、导演分镜草图、低精度面部和轻灰阴影。
10. 是否禁止照片感、电影剧照感、真实皮肤、精修五官、复杂光影和写实摄影。
11. 是否执行了 `最终提示词清洗规则`，删除 `realistic cinematic framing`、`photorealistic`、`film still`、`real skin texture`、`highly detailed face`、`cinematic lighting` 等仿真导向正向词；固定否定句除外。
12. 是否禁止画内文字。
13. 是否没有新增角色、道具、剧情或改写对白。
14. 参考图是否被转成具体画面控制信息。
15. 用户提供俯视图、平面图、站位图或简笔画平面图时，是否输出了 `俯视平面图锚定表`。
16. 是否明确写出俯视平面图优先于透视场景参考图；冲突时以俯视图为准。
17. 同一场景连续九宫格是否输出了 `机位编号表`，并且每格都绑定一个主机位编号。
18. 同一机位服务多个格子时，是否只允许裁切或缩放，不允许重新摆放家具和固定陈设。
19. 生图时是否输出 ZIP，且 ZIP 内为 `prompts.md` 加 `pages/*.png`。
20. 是否完整阅读了全部剧情、分镜表、截图文字或可见表格内容后再规划宫格。
21. 是否先建立 `空间概念`，锚定方向、摄影机轴线、初始坐标、允许位移和禁止漂移。
22. 有来源表格、截图或剧情文本时，是否输出了 `空间概念`。
23. 有来源表格、截图或剧情文本时，是否输出了 `主体建筑朝向表`。
24. 室内场景是否输出了 `室内平面布局表`。
25. 同一场景连续九宫格是否输出了 `前序宫格布局继承表`。
26. 有来源表格、截图或剧情文本且场景存在门槛/内外边界时，是否输出了 `门槛边界表`。
27. 同一对象在多个格子可见时，是否输出了 `对象足迹与朝向表`。
28. 有来源表格、截图或剧情文本时，是否输出了 `摄影机朝向表`。
29. 有来源表格、截图或剧情文本时，是否输出了 `对象可见性表`。
30. 有来源表格或截图时，是否输出了 `九格取舍表`。
31. 是否完成对象可见性、足迹朝向、室内平面布局、前序宫格继承和边界/机位内部冲突检查，避免同一对象同一格既被允许可见又被要求画外，或同一对象多格世界坐标/朝向不一致，或家具陈设多格平面位置不一致，或后序格重建空间，或机位朝向与背景内容矛盾。
32. 生图时是否完成 PNG 目检，并在不合格时收紧提示词重生。
