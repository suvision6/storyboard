## Prompt列详细指南 (视频生成提示词)

**适用范围**：仅用于 Seedance 2.0 Prompt 列生成，不处理其他模型降级。

Prompt 列为每个镜头输出可直接用于 AI 视频模型的结构化提示词。按 Seedance 2.0 规则输出。不要在本 reference 中加入其他模型格式。

---

### Seedance 2.0 输入限制

| 输入类型 | 数量上限 | 支持格式 | 大小限制 |
|----------|----------|----------|----------|
| 图片 | ≤ 9 张 | jpeg、png、webp、bmp、tiff、gif | 每张 < 30 MB |
| 视频 | ≤ 3 个 | mp4、mov | 每个 < 50 MB，总时长 2–15s |
| 音频 | ≤ 3 个 | mp3、wav | 每个 < 15 MB，总时长 ≤ 15s |
| 文本 | 自然语言提示词 | — | — |
| **总文件数** | **≤ 12 个** | — | — |

**注意事项**：
- **不支持写实真人脸部素材**（图片和视频均不可），系统会自动拦截。
- 有参考视频时生成费用略高。
- 优先上传对画面或节奏影响最大的素材。

---

### Seedance 2.0 @引用系统

Seedance 2.0 通过 `@` 指定每个素材的用途：

```
@图片1    @图片2    @图片3   ...
@视频1    @视频2    @视频3
@音频1    @音频2    @音频3
```

**为每个引用明确指定用途**：

| 用途 | 示例写法 |
|------|----------|
| 首帧 | `@图片1 作为首帧` |
| 尾帧 | `@图片2 作为尾帧` |
| 人物形象 | `参考 @图片1 的人物形象` |
| 场景/背景 | `场景参考 @图片3` |
| 运镜 | `参考 @视频1 的运镜效果` |
| 动作 | `参考 @视频1 的动作编排` |
| 特效 | `完全参考 @视频1 的特效和转场` |
| 节奏/节拍 | `视频节奏参考 @视频1` |
| 音色/语气 | `旁白音色参考 @视频1` |
| 背景音乐 | `背景BGM参考 @音频1` |
| 音效 | `音效参考 @视频3 的音效` |
| 服装 | `穿着 @图片2 的服装` |
| 产品外观 | `产品细节参考 @图片3` |

**多引用组合示例**：
```
@图片1 的人物作为主体，参考 @视频1 的运镜和动作编排，
背景BGM参考 @音频1，场景参考 @图片2
```

### 参考图处理规则 (Prompt列专用)

当用户提供参考图时，必须先理解再写提示词。参考图信息**不能只写"参考图""如图所示"**，必须转成具体文字描述。

**角色参考图** → 提取并写入：外观特征（脸型、发型、五官识别点、体态）、服装特征（类型、颜色、款式、配饰）。同一角色在所有分镜中保持同一外貌、服装、体态。

**场景参考图** → 提取并写入：空间结构（室内/室外、门窗、走廊、通道）、环境特征（材质、陈设、空间大小）、氛围光影（整体感受，不写复杂灯位/色温数值）。

**角色站位图** → 提取并写入独立段落：每个角色的位置、朝向、相互距离、互动方向。如果角色站位图和用户文字描述冲突，以角色站位图为主。站位线稿只用于位置/朝向/距离参考，不用于外观/场景/光影/服装等其他参考。

**道具参考图** → 提取并写入：道具形态、尺寸感、材质、颜色、关键识别点、如何被角色使用/触碰。

**未标注图片类型** → 优先根据内容自动判断；无法可靠判断时，保守转成"画面主体、空间关系、关键道具形态"的文字描述。

### Seedance 提示词结构模板

每个镜头的 Prompt 列输出遵循此结构：

```
时间：X秒-Y秒
景别：[简要角度 + 基础景别]
构图：[主体与同框角色/道具的简要位置关系]
运镜手法：[固定锁机/推进/拉远/环绕/横摇/纵摇/手持微晃/跟拍/摇臂升降/甩镜头]
画面内容：[主体动作、表情情绪、道具互动、声音台词、动作结果]
```

**景别保持轻量**：`[平视中景。]` `[微仰拍近景。]` `[高角度全景。]` — 不需要写精确角度数、复杂方位或摄影调度。

**运镜必须是真实摄影运动**：从标准运镜词库中选择，不得把"焦点锁定""焦点切换""失焦变清""剪辑切换""荷兰角"当成运镜手法。

**画面内容必须写清**：主体是谁、正在做什么、动作起因/过程/结果、角色情绪如何被看见（表情/身体姿态/手部动作/视线）、道具如何被触碰、对白原文（不得改字）。

### 15秒硬上限与拆段规则

Seedance 2.0 单次生成最大时长 **15 秒**。

- 单段输出时，最后一个分镜的结束时间不得超过 15 秒。
- 若镜头内容按自然节奏估算超过 15 秒，不得输出超长时间段（如 `时间：0秒-20秒`），必须**自动拆成多个 ≤15 秒的片段**。
- 每段内部时间从 0 秒重新开始。
- 优先按剧情节拍、场景变化、动作完成点、对白段落和情绪转折切分。
- 对白跨段时，按自然语义分到不同片段，必须完整保留原文，不得为了压缩时长改字或删句。

**时间估算参考**：
- 简单动作或单一反应：2-3 秒
- 普通动作 + 简短对白：3-4 秒
- 复杂动作、长对白、多人反应：4-6 秒
- 强动作段落（奔跑、打斗、跌倒）：5-8 秒

时间必须连续递增，不出现重叠或倒退。

### 镜头审校与静默重写

所有分镜在输出到 Prompt 列前，必须先审校是否服务剧情、站位、动作和 Seedance 2.0 可执行性。

**修正优先级（从高到低）**：
1. 对应类型的参考图（角色→外观服装、场景→空间结构光影、站位→位置朝向距离、道具→形态功能）
2. 剧情事实 / 对白原文
3. 空间连续和角色方位
4. 动作因果和情绪转折
5. 景别、构图、运镜建议

**必须触发修正的情况**：
- **景别与内容不匹配**：大特写却要表现多人站位、全景却要表现细微表情
- **运镜不可执行**：把焦点/失焦/景深变化写成运镜手法
- **前后镜头打架**：角色方位、视线方向突然变化，且剧情没有展示位移过程
- **镜头信息过载**：一个镜头塞入过多动作、多人反应、复杂走位
- **无意义镜头**：只写"沉默""保持不动"且没有动作/对白/情绪转折/信息增量

**修正策略**：
- 静默修正，不输出"我改了哪里"
- 优先保留原镜头数量
- 不得改变剧情事实、不得删除或改写对白、不得新增角色/道具/剧情

---

### 本镜头可见角色判定

写每个镜头的 Prompt 前，先静默判断"本镜头实际出镜角色名单"。

- 剧情段落中出现过的人，**不等于本镜头都出镜**。
- 特写、近景、中近景或单人反应镜头，只写实际拍到的角色。
- 不出镜角色不得写身体动作、表情、站位、反应或遮挡关系。
- 多人镜头中，单镜头最多重点跟踪 **3 个角色**。超过 3 个可见角色时，只点名核心 2-3 人，其他人用"背景人群"轻量处理。
- 多人镜头中，除主体外的可见角色也必须有轻量、明确、可见的状态或反应（如视线转向说话者、手指收紧、身体后退半步）。

---

### 连续性规则

- 同一角色在所有分镜中保持同一外貌、服装、体态和身份。
- 同一场景在所有分镜中保持空间结构一致。
- 同一道具在所有分镜中保持形态、位置和功能连续。
- 角色站位、朝向、距离必须连续；发生位移时，必须在分镜中展示移动过程。
- 不得让角色凭空出现在新位置，不得让道具凭空消失或改变外观。
- 切换镜头后，仍要重新交代必要的角色位置和朝向。
- 前后镜头不得互相打架。

---

### 声音与对白规则

- 对白必须**原样保留**，不增删改任何一个字。
- 必须写清谁说话、对谁说话。
- 环境声、机械声、脚步声、撞击声、电子音、颤音等必须有来源。
- 音色或听感可以写，但不能凭空添加用户未提供的异常声音质感。
- 如果用户明确写了声音风格（如"电子机械感，人工智能，男性音色"），必须保留。

---

### 禁令与禁词

**以下内容不得出现在 Prompt 列输出中**：

- "如图所示""严格参考图片""根据站位参考图""参考图中显示""图中显示"等元话术
- "已修正""原分镜问题""修改原因""优化说明"等审校解释
- 改写、删除或省略用户原始对白
- 编造新角色、新道具、新剧情
- 凭空添加具体家具、建筑结构、道具颜色/数字/纹样/品牌/材质/功能
- 凭空添加血丝、猩红、泪痕、伤口、哭腔、嗓音发沙等异常生理/声音质感
- 使用"仿佛""犹如""宛如""好似"等比喻词
- 写不可见心理活动、气味、触觉、温度感受
- 自我过肩（如"镜头越过A的肩膀拍摄A"）
- 无信息反应（只写"听完""沉默""保持不动""看着对方"）
- 多人镜头只写主体动作、遗漏其他可见角色的状态
- 把同场景但本镜头不出镜的角色写进构图或画面内容

**禁词（任何情况下不得出现）**：

`令人叹为观止` `令人惊叹` `令人着迷` `精心打造` `匠心独运` `独具匠心` `视觉盛宴` `光影交响` `完美呈现` `极致体验` `引人入胜` `震撼人心` `巧妙融合`

---

### Prompt列输出前自检

1. 是否按 Seedance 格式输出（时间/景别/构图/运镜手法/画面内容）
2. 是否遵循了 15 秒上限，需要拆段时是否正确拆段
3. 对白是否原样保留，说话者和对象是否清楚
4. 参考图是否转成了具体文字，没有使用元话术
5. 景别是否轻量、运镜是否真实可执行
6. 画面内容是否只描述本镜头可见角色
7. 是否没有编造新角色/道具/剧情
8. 是否没有输出禁词和禁用表达
