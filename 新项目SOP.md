# 新项目SOP

## 1. 目标

保证每一个新项目都以统一方式创建、归档和执行，避免内容散落到电脑其他位置。

## 2. 核心规则

1. 每执行一个项目，必须先创建一个项目文件夹。
2. 文件夹命名格式统一为：
   - `project-编号-项目名称`
3. 所有相关内容必须只放在对应项目文件夹中。
4. 不允许把项目资料、笔记、产物、草稿散放到其他位置。

## 3. 新项目创建流程

1. 确认项目编号。
2. 确认项目名称。
3. 创建目录：
   - `project-编号-项目名称`
4. 在目录内创建必要子目录：
   - `assets/`
   - `notes/`
   - `outputs/`
5. 在目录内创建项目文档：
   - `01-需求说明.md`
   - `02-执行记录.md`
   - `03-交付说明.md`
6. 开始执行前，先确认相关信息都已收纳到该目录。

## 4. 项目目录标准

```text
project-001-项目名称/
  01-需求说明.md
  02-执行记录.md
  03-交付说明.md
  assets/
  notes/
  outputs/
```

## 5. 执行中的要求

1. 所有新产生的内容都先考虑是否属于当前项目。
2. 属于当前项目的内容，立刻放回项目文件夹。
3. 需要复用的方法，优先写进全局工作台或踩坑日志。
4. 项目结束时，确认目录内内容完整、整洁、可追溯。

## 6. 结束检查

1. 项目文件夹是否已创建。
2. 文件是否都在项目文件夹内。
3. 是否有遗漏到其他位置的内容。
4. 是否有可复用经验需要同步到全局复利与踩坑日志。
5. 是否有新的全局规则需要同步到全局工作台。
