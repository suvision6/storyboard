# Camera Language Reference

<!-- for skill-version: 2.4.5 / rule-revision: 2.4.5-director-shot-integrity-guard-2026-07-18 -->

仅在 **Gate A 最新轮次批准且 `reviewed_hash` 匹配、Beat 层冻结之后**阅读本文件，用于选择机位、景别、运镜和撰写 `运镜+主画面描述(含台词)` 列。禁止用本文件反向主导 Beat 拆分；发现 Beat 层问题必须申请回流至 Gate A，不得在摄影层自行变通。本文件无权修改任何受保护审计字段（清单见 `SKILL.md` 第 0 节），也不用于生成 Prompt 列。

## 核心原则

列 5 是主表核心列。所有描述必须是摄影机可以直接拍摄的视觉事实。

### 三要素强制格式

每个镜头必须以三元组开头：

```text
[摄影机视角/高度, 构图景别/特殊视角, 摄影机运动方式]
```

示例：

- `[微俯视, 全景, 摇臂下降]`
- `[平视, 中特写, 固定镜头]`
- `[微仰拍, 七分身镜头, 缓缓推轨]`
- `[平视, 过肩镜头, 固定镜头]`
- `[俯拍 -> 平视, 全景 -> 特写, 伸缩摇臂下降]`
- `[微俯视, 大全景, 伸缩摇臂缓慢下降]`
- `[高角度俯拍, 大远景, 航拍缓慢推进]`
- `[平视, 中景, 斯坦尼康跟随]`

若涉及过肩、带关系单人、主观视角等特殊视角，将其归入第二项。

### 场景首镜站位

每个场景第一个镜头必须保留 `【场景首镜站位】`，并用一句话明确交代所有可见互动角色的初始站位和朝向；没有可见互动角色时写空间边界、固定物方向或主观视觉锚点：

```text
【场景首镜站位】（A位于门口，面向室内；B站在桌边，面向A，两人隔桌相对。）
```

多人场景必须在首镜完成所有可见互动角色的基础位置、朝向、距离与主要视线关系。`multi_character` / `both` 的 `visible_characters` 至少 2 人，且首镜必须覆盖 `position` fact。第二镜只能补充遮挡、前后景或细微关系，不得补交基础站位；后续只在位置或朝向变化时写 `【站位位移】`。

### 空间与多人首镜锚定

每个场景必须先确认连续性台账中的 `first_shot_anchor_type`。类型为 `space`、`multi_character`、`both` 时，首镜必须使用全景类景别：`大远景`、`大全景`、`全景`、`中全景`。

- 室外大场景优先 `微俯视大全景` 或 `平视大全景`，先交代地形方向、入口/出口、危险来源和人物起点。
- `space`、`both` 类型的户外大场景首镜优先使用 `摇臂升降`、`伸缩摇臂`、`航拍推进`、`航拍下降`、`航拍横移`；不得在 `大远景` 或 `大全景` 中使用斯坦尼康。
- 室内多人对话优先 `平视全景` 或 `微俯视全景`，先交代门窗/桌椅/固定物、人物左右关系、朝向、距离和主要对视轴线。
- 多人场景首镜必须完成基础站位锚定，再进入正反打、特写、道具特写或单人反应；第二镜不得承担迟到的基础站位补交。
- `single_continuation`、`subjective` 可使用中景或中近景首镜，但仍必须保留 `【场景首镜站位】` 标签，在标签中写清空间边界、固定物方向或主观视觉锚点。

全景不是空镜。首镜必须承载原文中的空间、人物、行动或气氛信息，不得新增无剧情事实的过渡镜头。

### 相邻镜头信息增量

同场相邻镜头必须有明确画面信息增量，并由 `camera_geometry` 证明机位关系。每镜明确 AXxx 轴线、摄影机所在侧、相对位置/朝向、主体与屏幕方向。本场首镜为 `scene_start`；之后同轴 `same | minor` 表示没有达到等价 30 度机位变化，**改变景别不等于改变机位**，默认合并或重设。`substantial` 必须伴随不同 `camera_position`。

- 单一封闭空间、单人独处或 `single_continuation` 场景，不得用近景/中近景/嘴角特写连续切分同一段情绪。
- 台词后的笑、眼神、呼吸、嘴角、闭眼等短反应，若没有新的空间、道具、声音、位置或现实层级事实，应写在同一镜头内。
- 同一句台词被逗号、顿号、冒号或分号切开时，不得拆成相邻两镜；用同一镜内表演、停顿、轻微推拉、横摇或呼吸节奏承接。

**2.4.5 跳切例外**：只有明确有意的跳切可在后镜写 `[跳切说明]`，并由 Gate B 精确批准 `shot:<镜号>:jump-cut`，此时降为人工 WARN。旧 `[必拆相邻]` 只保留历史合同语义，不能绕过本版机位几何检查。

### 轴线与屏幕方向

- 同一 AXxx 从 `side_a` 直接切到 `side_b`（或反向）是 180 度越轴，默认 FAIL；只有后镜写 `[越轴说明]` 且 Gate B 批准 `shot:<镜号>:axis-cross` 才降 WARN。
- `axis-reversal` 仍只表示摄影机纵深推进/拉出方向反转，不是越轴，两者不得混用。
- 同一人物的 facing / eyeline / movement 从画左反到画右，或从向镜头反到离镜头，后镜必须有该人物 position/facing 的 `continuity_update`；否则 FAIL。
- 主观镜必须把 `pov_character` 映射到本场角色，非主观镜必须为 null。主画面提到的角色必须属于 visible、offscreen 或 POV。

### 纵深推拉方向反转

同场相邻镜头**默认禁止纵深推拉方向反转**：`推进`、`急推`、`推轨推进`、`缓慢推进`、`航拍推进` 后不得直接接 `拉出`、`拉远`、`推轨拉出`、`缓慢拉出`；反向组合同样禁止。这是镜头纵深运动方向规则，不是 180 度越轴；人物空间轴线与视线连续性仍由 `spatial_axis` 独立约束。

- 默认处理：若相邻镜头需要从压力转向距离感，第二镜改为固定、横移、环绕、垂直摇移、切换主体或切换景别。
- **逃生标注**：确有叙事动机（例如推进完成信息揭示后，拉出暴露人物孤立或空间代价）时，在第二镜备注写 `[反转动机]` 加一句话理由。校验器对有标注的组合降为 WARN，并自动列入 Gate B 复核清单；保留时须登记 `shot:<镜号>:axis-reversal`。该 token 仅为历史兼容机器名，不表示 180 度越轴；泛化批准无效，无标注的纵深推拉方向反转为 FAIL。
- `[反转动机]` 不得用于掩盖无动机的节奏偷懒；Gate B 驳回后必须改用默认处理方案。

### 画面描述逻辑链

```text
[摄影机视角与高度] -> [构图取景与景别] -> [摄影机运动方式] -> [主体动作与表演] -> [嵌入台词]
```

- 只写看得见的：形状、颜色、光线、阴影、速度、方向、身体动作、面部变化。
- 情绪表达用可见反应：瞳孔收缩、眉头紧皱、身体紧绷、身体颤抖、微微挑眉、深呼吸、下颌咬紧、转身、低头、抬头、手指收紧等。**此类视觉化词汇只允许出现在本列，不得回写进 `fact.text`**（见 `continuity-shot-data.md` 字面化规则）。
- 禁止使用比喻词：`仿佛`、`犹如`、`宛如`、`好似`。
- 禁止使用营销辞藻：`令人叹为观止`、`视觉盛宴`、`光影交响`、`完美呈现` 等。

### 情绪可拍化

- 只根据原文、当前 Beat/fact 或剧情中已成立的情绪设计表演；禁止自动增强冲突、添加情绪阶段或强行峰值失控。
- 每镜选择 1–3 个高信号可见/可听表现，优先从眼神/面部、呼吸/声音、姿态/手部中取样。不强制四类动作齐全；麻木、屏息、压抑、僵住等有依据的静止可直接保留。
- 人物位移、道具操作、VFX 或其他不可逆状态变化必须展示起点、过程、终点；普通微动作只写有叙事价值的可见结果。
- 说话者的音量、音质、语速和呼吸写在对白引号外。倾听者反应只能来自 `visible_characters`，且短反应原则上留在同一镜内。
- 脚步、衣物摩擦、接触碰撞等不改变剧情状态的必然反馈可随已有动作表达。破裂、掉落、移位、光源变化等状态反馈必须命中原文/fact，并在需要时登记 `continuity_updates`。
- 2.4.5 的非对白描述不得使用毫米/厘米/米、Hz/K、精确角度等执行数值。使用“一臂距离”“隔桌相对”“低角度”“冷暖倾向”等相对可见描述；镜头秒数、年龄、画幅比例与原样对白中的数值豁免。`same | minor | substantial` 是机器分级，不得伪造具体度数。
- Gate B 前生成情绪表演摘要，逐镜列出情绪依据、可见/可听表演、前后镜连续性、声音/环境反馈依据和保留/修改结论；该摘要不进入 JSON。

### 表演链与证据

- 同一人物动作→反应→台词默认写在同一镜头，先在 `performance_chains` 登记事实步骤。短反应、`recommended` 或插镜倾向不能独立构成拆分理由。
- 确因必拆事实、现实/时间/空间断裂或明确编辑强调而拆分时，后镜写 `chain_breaks`、`[动作链拆分]`，并由 Gate B 批准 `shot:<镜号>:performance-chain-break`。
- 有可见人物的每镜保留 1–3 项 `performance_cues`，从 gaze/face、breath/voice、posture/hand、stillness 选择；描述必须逐字存在于画面，并由本镜事实取证。
- 人物位移、道具操作、VFX 或连续性状态变化用 `action_progressions` 写清起点、过程、终点；三段都必须在主画面可见。
- 环境声、光或接触反馈承担信息时，用 `environment_cues` 标明 `fact` 或 `inevitable_action` 依据；不得把风格偏好伪装成事实。
- 相邻镜完全复用同一 source span 默认是重复拆镜。只有结构性不可合并时，后镜登记 `source_reuse`、`[原文复用]`，并由 Gate B 批准 `shot:<镜号>:source-reuse`。

### 台词嵌入

将台词融入动作尾端：

```text
A转向B，手指收紧，说："台词内容。"
```

多角色同场时明确说话对象。禁止脱离表演动作的纯说明句式。对白保真校验只比对引号内内容，嵌入句式与引号标点不计入"改字"，但**引号内的对白一字不得改**。

## 连续性铁律

- 每个镜头的开场状态必须继承上一镜头的结束状态。
- 角色位置、朝向、情绪状态、道具位置不允许无依据跳跃或重置。
- 位置或朝向变化（**含被动位移：被拖走、击飞、摔倒等**）必须在镜头中展示或明确承接变化过程，写 `【站位位移】`，并在 `continuity_updates` 登记有事实 ID 依据的迁移。迁移触发的完整判定以 `continuity-shot-data.md` 的固定触发表为唯一出处。
- 切换镜头后，只在画面关系确有需要时交代角色位置和朝向，不重复完整首镜站位。
- 前后镜头不得互相打架：同一角色的左右方位、视线方向、互动对象和移动方向必须能连续解释。

## 摄影机语言参考

### 摄影机视角与高度

| 术语 | 用法 |
| --- | --- |
| 平视 | 日常对话、客观观察 |
| 微俯视 | 压低人物力量、建立地面关系 |
| 高角度俯拍 | 展示空间、队形、陷落感 |
| 正俯拍 | 强调阵法、地面轨迹、包围关系 |
| 微仰拍 | 提升人物力量或神性压迫 |
| 低角度仰拍 | 强化威压、巨大物或被压迫反差 |
| 主观视角 | 明确角色正在看见什么 |

### 景别与特殊视角

| 术语 | 用法 |
| --- | --- |
| 大远景 | 建立地貌或宏观空间 |
| 全景 | 建立人物站位和动作范围 |
| 中景 | 对话、基本动作和人物关系 |
| 中近景 | 表情与动作并重 |
| 特写 | 关键表情、道具、伤口、手部动作 |
| 微距特写 | 极小道具、纹路、光点、瞳孔 |
| 过肩镜头 | 对话方向和权力关系 |
| 带关系单人 | 单人表演同时保留前景关系 |

### 摄影机运动

| 术语 | 用法 |
| --- | --- |
| 固定镜头 | 信息明确、压住情绪、避免多余运动 |
| 缓慢推进 | 信息揭示、心理压力加深 |
| 缓慢拉出 | 孤立人物、暴露空间代价 |
| 上升拉远 | 段落或全片视觉收束，让人物被空间吞没或离开主要行动点 |
| 横移跟拍 | 人物横向位移或空间发现 |
| 手持微晃 | 危机、身体不稳、现场混乱 |
| 摇臂升降 | 空间层级、坠落、大尺度空间，也可用于户外/室内大空间首镜建立 |
| 伸缩摇臂 | 从大全景压入人物、从人物拉出空间、建立大空间层级 |
| 斯坦尼康跟随 | 地面机位下的人物持续位移，适配小全景、全景偏人物、中景、中近景、近景 |
| 航拍推进/下降/横移 | 超大范围地貌、建筑全景、道路、大范围位移、俯拍空间关系 |
| 环绕半圈 | 权力压迫、困局、包围 |
| 垂直纵摇 | 从人物视线引向上方或下方信息 |

### 运镜与景别匹配

- `斯坦尼康` 只用于地面人物持续位移，不用于 `大远景`、`大全景`、超大户外空间、建筑整体建立、地貌建立或航拍俯视。
- `航拍` 必须服务于大范围空间关系，景别应为 `大远景`、`大全景` 或 `全景`，视角应为 `高角度俯拍`、`正俯拍`、`微俯视` 或其他明确俯拍视角。
- `摇臂`、`伸缩摇臂` 可用于空间首镜建立，也可用于从空间压入人物或从人物拉出环境。
- 室内多人对话首镜不默认航拍，优先使用固定镜头、缓慢横移、轻微推轨或摇臂微降。
- 三元组中的景别不得同时混写远景类和近景/特写类。需要从空间压入表情或从表情拉出空间时，必须使用 `光学变焦`、`快速推拉`、`急推`、`急拉`、`推轨推进/拉出`、`伸缩摇臂` 等能完成跨度的运镜，并在备注写 `[景别跨度]` 理由；否则拆为信息明确的两个镜头或选择单一景别。

运镜选择必须服务于当前镜头的信息增量。**不得为了消除校验警告而机械替换三元组**；任何运镜变化都应能解释为画面信息或情绪节奏需要，必要时记入 `revision_log`。

## 本文件涉及的备注标注

本文件规则会产生以下备注标注（均在 `SKILL.md` 第 8 节封闭清单内）：`[景别跨度]`、`[必拆相邻]`、`[反转动机]`。使用清单外标注视为过程痕迹，FAIL。
