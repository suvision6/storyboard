# Project Skill Routing

Use `skills/su-fenjingskill-zh` as the only storyboard skill for full screenplay-to-shot-list work.

When a task needs a complete 7-column storyboard table, use `su-fenjingskill-zh` exclusively. It owns the table structure, shot numbers, scene mapping, original script mapping, shot duration, camera/main-image description, notes, Prompt column, shot_data JSON, Markdown delivery, Excel delivery, and validation.

Do not route Prompt-column work to other skills. The Prompt column is generated by `su-fenjingskill-zh` under its current rules.

Keyframe is outside the current contract. Do not generate a Keyframe column or a `shots[*].keyframe` field. Do not route role-position continuity work to other skills; continuity handling remains owned by `su-fenjingskill-zh`.

If rules conflict, follow this priority:

1. The user's explicit instruction in the current turn.
2. `su-fenjingskill-zh` as the sole primary workflow and table contract.

Keep storyboard generation independent inside `su-fenjingskill-zh`. Do not merge, import, or apply other skill rules unless the user explicitly requests a separate one-off comparison.
