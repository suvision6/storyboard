# Image2 九宫格分镜提示词

## 版本

<!-- skill-version: 1.0.0 -->

当前版本：1.0.0。后续每次优化前，先保留当前版本；优化后递增版本号，并使用 Git 标签 `su-image2-storyboard-grid-zh-v{版本号}` 保存历史版本。

把 `su-fenjingskill-zh` 的分镜主表、局部镜号、某场戏，或用户提供的纯剧情文本，整理成适用于 Image2 的固定 3x3 九宫格黑白分镜图提示词。

本 skill 是独立下游交付工具。它只读取主表或剧情，不回写、不修订、不替换 `su-fenjingskill-zh` 的任何规则。

## 核心边界

- 不修改 `su-fenjingskill-zh`。
- 不回写主表。
- 不改变镜号、场景、原剧本段落、镜头时长、运镜主画面、备注、Prompt 列或故事板列。
- 不替换 `su-fenjingskill-zh` 现有“每镜单帧故事板提示词”规则。
- 不把九宫格、多宫格、站位图或 ZIP 交付规则写入主表。
- 如果需要和主表关联，只在本 skill 的 Markdown 中写“来源镜号/镜号范围”。

## 默认交付

默认生成一个 Markdown 提示词文件，而不是只在聊天中输出长文本。

文件名建议：

```text
image2_storyboard_grid_prompts_[项目名或场次名].md
```

如果用户没有指定保存位置，保存到当前任务的用户可见输出目录；如果环境没有输出目录，则保存在当前工作目录下的 `outputs/` 或用户指定目录。

Markdown 文件按镜号、镜号范围或场景组织，每段都是一条可复制到 Image2 的九宫格提示词。

## 输入优先级

按以下优先级处理输入：

1. 用户本轮明确要求。
2. 用户提供的美术资产或参考图。
3. `su-fenjingskill-zh` 主表中的镜号、场景、原剧本段落、运镜主画面、备注、Prompt、故事板列。
4. 纯剧情文本。

用户提供主表时，主表是剧情事实来源。用户只提供剧情时，允许从剧情推演九个画面节点，但不得新增剧情事实。

## 资产规则

用户提供美术资产或参考图时必须参考，并转成具体画面控制信息。

- 角色图控制：外貌、服装、体态、身份识别点。
- 场景图控制：空间结构、门窗、通道、家具、纵深、边界、环境基调。
- 道具图控制：形态、尺寸、材质、功能、被触碰或使用的方式。
- 站位图或尾帧控制：位置、朝向、距离、初始空间关系、上一段结束状态。
- 文字剧情控制：剧情事实、动作因果、对白原文、情绪变化、关键结果。

不要写“如图所示”“严格参考图片”“根据图片”等元话术。必须把图片信息转写成具体、可生成的画面描述。

### 资产映射确认

当图片用途不清、编号无法对应、参考图之间冲突，或用户文字与站位图/尾帧/场景图冲突时，只输出资产映射确认并停止，不生成正式提示词。

使用这个格式：

```markdown
资产映射确认：
我需要先确认图片和资产用途，避免把角色外观、场景结构、道具、站位图和尾帧混在一起。

| 文件或编号 | 我识别为 | 类型 | 控制内容 | 状态 | 备注 |
|---|---|---|---|---|---|
| ... | ... | 角色图/场景图/道具图/站位图/尾帧/未知 | ... | 已匹配/需要确认/冲突 | ... |

请回复“确认资产”，或逐条告诉我修改。
```

确认前不生成 Markdown 提示词文件，不生成 PNG，不生成 ZIP。

## Markdown 文件结构

默认文件结构：

```markdown
# Image2 九宫格分镜提示词｜项目名或场次名

## 镜号 001

来源：...

资产控制：
...

Image2 可复制提示词：
...

## 镜号 002-004

来源：...

资产控制：
...

Image2 可复制提示词：
...
```

标题规则：

- 单镜：`## 镜号 001`
- 连续镜号：`## 镜号 002-004`
- 场景范围：`## 场景 12-1｜镜号 001-009`
- 纯剧情无镜号：`## 剧情段 001`

每段必须包含：

- `来源`：来自主表镜号、场景、用户指定片段或纯剧情。
- `资产控制`：仅在用户提供参考图时输出；没有资产则省略。
- `Image2 可复制提示词`：一段完整、干净、可直接复制的九宫格提示词。

## Image2 提示词结构

每段 `Image2 可复制提示词` 使用中文为主，写成一段完整提示词。Markdown 中可以用 `格 1` 到 `格 9` 帮用户阅读，但必须明确这些编号不是画内文字。

提示词必须包含：

- 固定 3x3 九宫格。
- 每格都是 16:9 分镜画面。
- 黑白导演手绘线稿。
- 允许轻灰阴影辅助体积和空间。
- 同一角色身份在出镜时保持外貌、服装、体态和识别点一致；每格只呈现该画面实际需要的角色，不强制所有角色都出现在每一格。
- 同一场景、时间段、空间结构和线稿风格保持一致。
- 只有动作、表情、调度、构图、景别和观察角度可随剧情变化。
- 图中不要出现任何文字、镜号、角色名、对白、字幕、标注、说明框、箭头文字或水印。

推荐提示词骨架：

```text
生成一张 3x3 九宫格黑白导演手绘线稿分镜图，每格为 16:9 画面，整体为清晰九宫格排版。画面只使用黑白线稿和少量轻灰阴影。每个角色只在剧情需要出镜的格子里出现；同一角色一旦出镜，必须保持同一外貌、服装、体态和身份识别点。所有格子保持同一场景、同一时间段、连续空间结构和统一线稿风格。图中不要出现任何文字、镜号、角色名、对白、字幕、标注、说明框、箭头文字或水印。

格 1：...
格 2：...
...
格 9：...

九格按剧情推进排列，每格只承担一个清晰画面节点；不得新增角色、道具或剧情；角色位置、朝向、距离、道具归属和情绪状态必须连续。
```

## 九格内容规则

有 `su-fenjingskill-zh` 主表时：

- 按用户指定场景、镜号范围或若干镜头生成。
- 不重新拆主表，不改变镜号含义。
- 九格从主表里选择关键画面：开场关系、主要动作、道具变化、关键对白瞬间、情绪转折、信息揭示、冲突高潮、收尾状态。
- 若主表镜头少于九个，用同一镜头内的不同可见阶段、不同景别、不同角度、关键反应或道具细节补足。
- 若主表镜头多于九个，只保留最关键的九个视觉节点，不删改主表事实。

没有分镜表时：

- 从剧情文本推演九个可见画面节点。
- 不为了凑满九格新增角色、新道具、新剧情。
- 节点不足时，用同一剧情状态的不同景别、角度、构图、角色反应、道具细节或空间观察方式补足。
- 节点过多时，优先保留开场关系、主要动作、关键对白瞬间、道具变化、情绪转折、冲突高潮和收尾状态。

每格必须写清：

- 主体是谁。
- 主体在做什么。
- 景别与观察角度。
- 构图与空间关系。
- 角色朝向、距离和互动对象。
- 情绪如何被看见：眼神、嘴唇、下颌、呼吸、姿态、停顿、手部动作。
- 道具位置、归属、变化或使用方式。
- 动作结果或画面状态。

## 角色出镜规则

九宫格的连续性目标是“同一身份不变”，不是“每格出现同一批角色”。

- 单人镜头：只写当前格实际出镜的角色；其他角色可通过视线方向、画外声或空间压力存在，但不要画进画面。
- 双人镜头：写清两人的朝向、距离、谁在前景/后景、谁正脸可见、谁侧脸或背影可见。
- 三人镜头：优先建立三人的空间关系；如果画面过载，保留核心两人，第三人可放在远处背景或下一格交代。
- 群像镜头：只重点跟踪 1 到 3 个核心角色，其余人写成背景人群或队列，不逐一刻画身份。
- 正反打或特写：不要为了“角色一致”把未出镜角色硬塞进画面。
- 不允许同一角色在同一格里被复制成两个人；角色离开画面后，除非剧情显示其回到画面，否则后续格不强行出现。
- 角色外貌、服装、体态和道具归属只在该角色出镜时保持一致。

## 画面风格

默认风格固定为：黑白导演手绘线稿 + 轻灰阴影。

不要使用彩色电影感、厚涂插画、写实剧照、CG 游戏感、漫画对白框、海报构图、复杂纹理渲染或过度光影。九宫格分镜图的第一目标是清楚表达连续空间、动作和情绪节点。

## 画内文字禁令

默认禁止任何画内文字。不要要求 Image2 在图片里生成：

- 镜号
- 格号
- 角色名
- 台词
- 字幕
- 旁白
- 说明文字
- 方位标注
- 箭头文字
- 水印

Markdown 里的 `格 1` 到 `格 9` 只用于阅读和复制，不是画面内容。

## 生图追加规则

只有当用户明确追加以下需求时，才进入生图流程：

- 生图
- 批量生成
- 按这个 Markdown 生成图片
- 按文件内提示词生成宫格图
- 输出 ZIP

生图时：

- 读取已生成 Markdown 文件内每段 `Image2 可复制提示词`。
- 每段提示词生成一张独立 3x3 九宫格 PNG。
- 不合并成单张超长图。
- 不生成多页 PNG。
- 所有 PNG 打包成一个 ZIP 文件。

ZIP 固定结构：

```text
prompts.md
pages/
  shot-001_grid.png
  shots-002-004_grid.png
  scene-12-1_shots-001-009_grid.png
```

PNG 命名规则：

- 单镜：`shot-001_grid.png`
- 连续镜号：`shots-001-004_grid.png`
- 场景范围：`scene-12-1_shots-001-009_grid.png`
- 纯剧情段：`story-segment-001_grid.png`

每张 PNG 必须能独立使用、删除、替换或重新生成，不依赖其他 PNG。

## 输出纪律

- 默认先生成 Markdown 提示词文件。
- 不输出多个版本，除非用户明确要求。
- 不输出解释性总结到 Markdown 文件正文里。
- 不把内部推理节点、检查清单或修改说明写入可复制提示词。
- 不推荐其他模型或平台，除非用户明确询问。
- 不生成 JPG，除非用户明确要求。
- 不将 ZIP 作为默认提示词交付；ZIP 只用于用户追加生图后的图片交付。

## 输出前自检

在交付前静默检查：

1. 是否没有修改或要求修改 `su-fenjingskill-zh`。
2. 是否默认输出 Markdown 文件。
3. Markdown 是否按镜号、镜号范围、场景或剧情段组织。
4. 每段是否有一条完整可复制的 Image2 九宫格提示词。
5. 是否固定 3x3 九宫格，每格 16:9。
6. 是否固定黑白导演手绘线稿和轻灰阴影。
7. 是否禁止画内文字。
8. 是否没有新增角色、道具、剧情或改写对白。
9. 参考图是否被转成具体画面控制信息。
10. 生图时是否输出 ZIP，且 ZIP 内为 `prompts.md` 加 `pages/*.png`。
