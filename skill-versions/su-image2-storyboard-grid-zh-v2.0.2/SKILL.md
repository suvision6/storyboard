# Image2 九宫格分镜提示词路由

## 版本

<!-- skill-version: 2.0.2 -->

当前版本：2.0.2。此 skill 是兼容旧调用的路由入口，不直接承载完整生成规则。执行入口 `su-image2-storyboard-grid-ref-zh` 与 `su-image2-storyboard-grid-text-zh` 已强化第 1 格主平面锚定、后续宫格空间继承、对视轴线、反打肩位和银幕左右锁定。

## 核心边界

- 不修改 `su-fenjingskill-zh`。
- 不回写主表。
- 不改变镜号、场景、原剧本段落、镜头时长、运镜主画面、备注、Prompt 列或故事板列。
- 不替换 `su-fenjingskill-zh` 的每镜单帧故事板规则。
- 不把九宫格、多宫格、站位图或 ZIP 交付规则写入主表。
- 默认只生成 Markdown 提示词文件；只有用户明确要求“生图、批量生成、按 Markdown 生成图片、输出 ZIP”时才进入生图流程。

## 路由规则

先完整阅读用户本轮输入，再按以下规则选择唯一执行 skill。

### 转用参考图版

打开并遵守 `../su-image2-storyboard-grid-ref-zh/SKILL.md`，当输入包含任一情况：

- 场景参考图、角色参考图、道具参考图、美术资产图。
- 俯视图、平面图、站位图、机位图、镜头方向图。
- 尾帧、上一镜输出图、图片编号、资产路径、图片附件。
- 用户同时提供文字和参考图。

参考图版必须先审查资产/空间一致性。图片用途不清、编号无法对应、参考图互相冲突、图文空间冲突时，必须输出 `任务失败：参考资产冲突` 并停止，不生成正式提示词。

### 转用纯文字版

打开并遵守 `../su-image2-storyboard-grid-text-zh/SKILL.md`，当输入只包含任一情况：

- 剧本文字或纯剧情段落。
- 文字版分镜、Markdown 表格、Excel 表格内容。
- 分镜表截图或 Excel 截图的可见文字内容，需要先人工转写成结构化来源。
- 局部镜号、场景号、镜号范围，但没有任何参考图或资产图。

纯文字版不得使用参考图规则，不输出资产控制表，不因无图而要求用户补图。

## 输出语言

- Markdown 分析表、来源说明、空间锚定、取舍表使用中文。
- 最终 `Image2 可复制提示词` 使用英文主导。
- 角色名可写拼音或英文身份描述，例如 `Lin Xiaotong, a young Chinese woman`。
- 画内禁止任何文字；Markdown 中的格号和角色名只供用户阅读，不是画面内容。

## 执行纪律

- 本入口只做路由，不混用两个子 skill 的规则。
- 如任务已经明确属于某个子 skill，直接按子 skill 执行，不再重复解释路由。
- 若用户明确点名本入口但输入类型清楚，仍按上述规则分流。
- 若输入类型确实无法判断，优先检查是否存在任何图片/资产线索；有则参考图版，无则纯文字版。
- 两个执行入口都要求第 1 格作为主平面锚定格；后续格只能从第 1 格裁切、推进、反打、俯拍或换侧面角度，不得重新布置室内/外景空间。
