# Project Skill Routing

Use `skills/su-fenjingskill-zh` as the only storyboard skill for full screenplay-to-shot-list work.

When a task needs a complete storyboard table, use `su-fenjingskill-zh` exclusively. It owns the stable 7-column table structure, shot numbers, scene mapping, original script mapping, shot duration, camera/main-image description, notes, Prompt column, Markdown delivery, Excel delivery, JSON source data, and validation.

Do not route Prompt-column work to other skills. The Prompt column is generated by `su-fenjingskill-zh` under its current simplified five-field rules, after the director main table is locked.

Do not route role-position continuity work to other skills. Continuity logs, `continuity_updates`, scene first-shot positioning, and position/facing migrations are generated and validated by `su-fenjingskill-zh` under its current rules.

The primary storyboard contract no longer includes a keyframe/storyboard image-prompt column. If keyframe prompts are needed, they must be requested as a separate one-off artifact and must not modify the primary 7-column table, Prompt column, JSON, Markdown, Excel, or validation contract.

If rules conflict, follow this priority:

1. The user's explicit instruction in the current turn.
2. `su-fenjingskill-zh` as the sole primary workflow and table contract.

Keep storyboard generation independent inside `su-fenjingskill-zh`. Do not merge, import, or apply other skill rules unless the user explicitly requests a separate one-off comparison.
