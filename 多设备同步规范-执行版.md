# 多设备同步规范 - 执行版

## 适用对象

给 Codex 或其他 agent 直接执行。

## 目标

把当前 Mac 定义为主工作机，把一台 PC 定义为辅助机，把 GitHub 定义为唯一规则中枢，把 `~/.codex/skills` 定义为本机技能运行目录。

## 执行顺序

1. 先读取 `Agent.md`。
2. 再读取 `全局工作台.md`。
3. 再读取 `全局复利与踩坑日志.md`。
4. 再读取 `新项目SOP.md`。
5. 再读取 `多设备同步规范.md`。
6. 如任务涉及 skill，确认本机 `~/.codex/skills` 已存在对应目录。
7. 如任务涉及项目，确认项目目录已按 `project-编号-项目名称` 创建。
8. 如任务涉及规则修改，优先更新 GitHub 仓库版本，再同步本机可见入口。

## 必须执行

1. 新规则先在 Mac 上确认。
2. 长期规则必须写入 GitHub 仓库。
3. 技能必须安装到本机 `~/.codex/skills`。
4. 工作区如需要可见入口，可以使用软链接或索引文件。
5. 新项目必须先建目录，再放内容。
6. 项目内容只允许放入对应项目目录。

## 不得执行

1. 不得把临时文件、缓存、下载内容当作规则源。
2. 不得让 Mac 和 PC 互相盲目覆盖。
3. 不得把未确认草稿直接同步为主版本。
4. 不得把本机临时状态写成全局规则，除非先更新仓库。
5. 不得把项目内容散落到工作区外的其他位置。

## 冲突处理

当多个位置内容不一致时，按以下顺序判定：

1. GitHub 仓库
2. Mac 本机确认版本
3. PC 本机版本

如果冲突无法自动消解：

- 暂停写入
- 先更新规则文档
- 再重新同步

## 检查清单

执行完成前必须确认：

1. 仓库版本已更新或确认无需更新。
2. Mac 上能读到对应文档或技能。
3. PC 只承担执行端角色。
4. `~/.codex/skills` 中存在对应技能。
5. 工作区入口可见。
6. 没有临时文件泄漏到仓库。
