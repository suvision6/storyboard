# 全局复利与踩坑日志

## 1. 用途

这份文档专门记录：

- 犯过的错误
- 踩过的坑
- 走过的弯路
- 已经验证有效的方法
- 可持续复用的做法

## 2. 记录原则

1. 只记录真正有复用价值的经验。
2. 记录时尽量写清楚：
   - 发生了什么
   - 为什么会发生
   - 以后怎么避免
   - 以后怎么复用
3. 遇到重复问题，优先查这份文档。
4. 发现新坑，及时补充。

## 3. 记录模板

```text
日期：
项目：
问题/坑：
表现：
根因：
影响：
解决办法：
预防方法：
可复用方法：
状态：已验证 / 来自记忆 / 我的推断
```

## 4. 可持续复用方法

- 先查历史，再动手
- 把常见流程写成 SOP
- 把重复修正写成清单
- 把稳定套路写成模板
- 把高频失误写成预检项

## 5. 日志使用规则

1. 新发现的坑，先记录，再继续做事。
2. 反复出现的坑，必须升级成流程或检查项。
3. 一旦某个方法稳定有效，就标记为可复用。
4. 若记录和后续事实冲突，以最新验证结果为准。
