# Camera Language Reference

Use this reference only after Beat IDs and the non-camera director fields are locked, when selecting camera angle/shot size/movement or writing the `运镜+主画面描述(含台词)` column. Do not use it to split Beats or generate the Prompt column.

## 核心原则

列 5 是主表核心列。所有描述必须是摄影机可以直接拍摄的视觉事实。

### 三要素强制格式

每个镜头必须以三元组开头：

```text
[摄影机视角/高度, 构图景别/特殊视角, 摄影机运动方式]
```

示例：

- `[微俯视, 全景, 摇臂下降]`
- `[平视, 中特写, 固定镜头]`
- `[微仰拍, 七分身镜头, 缓缓推轨]`
- `[平视, 过肩镜头, 固定镜头]`
- `[俯拍 -> 平视, 全景 -> 特写, 伸缩摇臂下降]`
- `[微俯视, 大全景, 伸缩摇臂缓慢下降]`
- `[高角度俯拍, 大远景, 航拍缓慢推进]`
- `[平视, 中景, 斯坦尼康跟随]`

若涉及过肩、带关系单人、主观视角等特殊视角，将其归入第二项。

### 场景首镜站位

每个场景第一个镜头必须用一句话明确交代所有出场角色的初始站位和朝向：

```text
【场景首镜站位】（A位于门口，面向室内；B站在桌边，面向A，两人相距约三米。）
```

后续镜头不重复完整站位，只在位置或朝向发生变化时写 `【站位位移】`。

### 空间与多人首镜锚定

每个场景必须先判断 `first_shot_anchor_type`。类型为 `space`、`multi_character`、`both` 时，首镜必须使用全景类景别：`大远景`、`大全景`、`全景`、`中全景`。

- 室外大场景优先 `微俯视大全景` 或 `平视大全景`，先交代地形方向、入口/出口、危险来源和人物起点。
- `space`、`both` 类型的户外大场景首镜优先使用 `摇臂升降`、`伸缩摇臂`、`航拍推进`、`航拍下降`、`航拍横移`；不得在 `大远景` 或 `大全景` 中使用斯坦尼康。
- 室内多人对话优先 `平视全景` 或 `微俯视全景`，先交代门窗/桌椅/固定物、人物左右关系、朝向、距离和主要对视轴线。
- 多人场景进入正反打、特写、道具特写或单人反应前，必须先有全景或中全景锚定。
- `single_continuation`、`subjective` 可使用中景或中近景首镜，但仍要写清空间边界、固定物方向或主观视觉锚点。

全景不是空镜。首镜必须承载原文中的空间、人物、行动或气氛信息，不得新增无剧情事实的过渡镜头。

### 相邻镜头信息增量

同场相邻镜头必须有明确画面信息增量。若可见主体相同、人物没有站位或朝向迁移、视角族相同、景别等级相近，且运镜只是固定、缓慢、轻微或呼吸式变化，应合并为一镜，或重新设计为能交代新事实的镜头。

- 单一封闭空间、单人独处或 `single_continuation` 场景，不得用近景/中近景/嘴角特写连续切分同一段情绪。
- 台词后的笑、眼神、呼吸、嘴角、闭眼等短反应，若没有新的空间、道具、声音、位置或现实层级事实，应写在同一镜头内。
- 同一句台词被逗号、顿号、冒号或分号切开时，不得拆成相邻两镜；用同一镜内表演、停顿、轻微推拉、横摇或呼吸节奏承接。
- 同场相邻镜头禁止无动机轴向反转。`推进`、`急推`、`推轨推进`、`缓慢推进`、`航拍推进` 后不得直接接 `拉出`、`拉远`、`推轨拉出`、`缓慢拉出`；反向组合同样禁止。
- 若相邻镜头需要从压力转向距离感，第二镜应改为固定、横移、环绕、垂直摇移、切换主体或切换景别，而不是直接反向轴向运动。

### 画面描述逻辑链

```text
[摄影机视角与高度] -> [构图取景与景别] -> [摄影机运动方式] -> [主体动作与表演] -> [嵌入台词]
```

- 只写看得见的：形状、颜色、光线、阴影、速度、方向、身体动作、面部变化。
- 情绪表达用可见反应：瞳孔收缩、眉头紧皱、身体紧绷、身体颤抖、微微挑眉、深呼吸、下颌咬紧、转身、低头、抬头、手指收紧等。
- 禁止使用比喻词：`仿佛`、`犹如`、`宛如`、`好似`。
- 禁止使用营销辞藻：`令人叹为观止`、`视觉盛宴`、`光影交响`、`完美呈现` 等。

### 台词嵌入

将台词融入动作尾端：

```text
A转向B，手指收紧，说：“台词内容。”
```

多角色同场时明确说话对象。禁止脱离表演动作的纯说明句式。

## 连续性铁律

- 每个镜头的开场状态必须继承上一镜头的结束状态。
- 角色位置、朝向、情绪状态、道具位置不允许无依据跳跃或重置。
- 发生位置或朝向变化时，必须在镜头中展示移动过程，写 `【站位位移】`，并在 `continuity_updates` 登记有事实 ID 依据的迁移。
- 切换镜头后，只在画面关系确有需要时交代角色位置和朝向，不重复完整首镜站位。
- 前后镜头不得互相打架：同一角色的左右方位、视线方向、互动对象和移动方向必须能连续解释。

## 摄影机语言参考

### 摄影机视角与高度

| 术语 | 用法 |
| --- | --- |
| 平视 | 日常对话、客观观察 |
| 微俯视 | 压低人物力量、建立地面关系 |
| 高角度俯拍 | 展示空间、队形、陷落感 |
| 正俯拍 | 强调阵法、地面轨迹、包围关系 |
| 微仰拍 | 提升人物力量或神性压迫 |
| 低角度仰拍 | 强化威压、巨大物或被压迫反差 |
| 主观视角 | 明确角色正在看见什么 |

### 景别与特殊视角

| 术语 | 用法 |
| --- | --- |
| 大远景 | 建立地貌或宏观空间 |
| 全景 | 建立人物站位和动作范围 |
| 中景 | 对话、基本动作和人物关系 |
| 中近景 | 表情与动作并重 |
| 特写 | 关键表情、道具、伤口、手部动作 |
| 微距特写 | 极小道具、纹路、光点、瞳孔 |
| 过肩镜头 | 对话方向和权力关系 |
| 带关系单人 | 单人表演同时保留前景关系 |

### 摄影机运动

| 术语 | 用法 |
| --- | --- |
| 固定镜头 | 信息明确、压住情绪、避免多余运动 |
| 缓慢推进 | 信息揭示、心理压力加深 |
| 缓慢拉出 | 孤立人物、暴露空间代价 |
| 上升拉远 | 段落或全片视觉收束，让人物被空间吞没或离开主要行动点 |
| 横移跟拍 | 人物横向位移或空间发现 |
| 手持微晃 | 危机、身体不稳、现场混乱 |
| 摇臂升降 | 空间层级、坠落、神殿尺度，也可用于户外/室内大空间首镜建立 |
| 伸缩摇臂 | 从大全景压入人物、从人物拉出空间、建立神殿/洞窟/山坳等空间层级 |
| 斯坦尼康跟随 | 地面机位下的人物持续位移，适配小全景、全景偏人物、中景、中近景、近景；用于穿行、跟走、绕行、贴近人物移动 |
| 航拍推进/下降/横移 | 超大范围地貌、建筑全景、高速道路、大范围位移、俯拍空间关系；适配大远景、大全景、全景和俯拍视角 |
| 环绕半圈 | 权力压迫、困局、包围 |
| 垂直纵摇 | 从人物视线引向上方或下方信息 |

### 运镜与景别匹配

- `斯坦尼康` 只用于地面人物持续位移，不用于 `大远景`、`大全景`、超大户外空间、建筑整体建立、地貌建立或航拍俯视。
- `航拍` 必须服务于大范围空间关系，景别应为 `大远景`、`大全景` 或 `全景`，视角应为 `高角度俯拍`、`正俯拍`、`微俯视` 或其他明确俯拍视角。
- `摇臂`、`伸缩摇臂` 可用于空间首镜建立，也可用于从空间压入人物或从人物拉出环境。
- 室内多人对话首镜不默认航拍，优先使用固定镜头、缓慢横移、轻微推轨或摇臂微降。
- 三元组中的景别不得同时混写远景类和近景/特写类。需要从空间压入表情或从表情拉出空间时，必须使用 `光学变焦`、`快速推拉`、`急推`、`急拉`、`推轨推进/拉出`、`伸缩摇臂` 等能完成跨度的运镜，并在备注写 `[景别跨度]` 理由；否则拆为信息明确的两个镜头或选择单一景别。

运镜选择必须服务于当前镜头的信息增量。不得为了消除校验警告而机械替换三元组；任何运镜变化都应能解释为画面信息或情绪节奏需要。
